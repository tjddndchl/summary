## HTTP 상태 코드 요약

### 1xx (정보)
- **100 Continue**: 서버가 요청의 일부를 받았으며, 나머지를 계속 전송해도 된다는 것을 알림.
- **101 Switching Protocols**: 서버가 클라이언트의 프로토콜 변경 요청을 수락함.

### 2xx (성공)
- **200 OK**: 요청이 성공적으로 처리됨.
- **201 Created**: 요청이 성공적으로 처리되었고, 새로운 리소스가 생성됨.
- **202 Accepted**: 요청은 접수되었으나, 아직 처리되지 않음.
- **204 No Content**: 응답에 컨텐츠가 없음. (예: DELETE 요청)
- **206 Partial Content**: 부분 컨텐츠 응답. 주로 범위 요청에 사용됨.

### 3xx (리다이렉션)
- **300 Multiple Choices**: 여러 옵션이 가능하며, 클라이언트가 선택해야 함.
- **301 Moved Permanently**: 리소스가 영구적으로 새 위치로 이동함.
- **302 Found (혹은 Moved Temporarily)**: 임시적으로 다른 위치로 리다이렉트.
- **304 Not Modified**: 이전에 방문한 리소스가 수정되지 않았을 때 사용.
- **307 Temporary Redirect**: 302와 유사하나, 요청 메서드가 변경되지 않음.

### 4xx (클라이언트 오류)
- **400 Bad Request**: 잘못된 요청.
- **401 Unauthorized**: 인증 필요.
- **403 Forbidden**: 권한 없음. (인증과는 별개)
- **404 Not Found**: 리소스를 찾을 수 없음.
- **405 Method Not Allowed**: 허용되지 않는 HTTP 메서드 사용.
- **406 Not Acceptable**: 서버가 요청의 주도적인 콘텐츠 협상 헤더에 정의된 허용 가능한 값 목록과 일치하는 응답을 생성할 수 없으며, 서버가 기본 표현을 제공하지 않음을 나타냅니다. (SSL 443 인증관련)
- **429 Too Many Requests**: 너무 많은 요청으로 인한 제한.

### 5xx (서버 오류)
- **500 Internal Server Error**: 서버 내부 오류.
- **501 Not Implemented**: 서버가 요청 메서드를 지원하지 않음.
- **502 Bad Gateway**: 하나의 서버가 다른 서버로부터 유효하지 않은 응답을 받았을 때.
- **503 Service Unavailable**: 서비스 일시 중지.
- **504 Gateway Timeout**: 게이트웨이 타임아웃 (다른 서버의 응답을 기다리는 동안).

## 크로스 오리진 (Cross-Origin) 오류

크로스 오리진 오류는 보안 목적으로 웹 브라우저에 의해 적용되는 **동일 출처 정책 (Same-Origin Policy, SOP)** 때문에 발생합니다. 이 정책은 웹 페이지의 스크립트가 다른 출처의 리소스에 접근하는 것을 제한합니다.

### 오류 발생 시나리오

1. 웹 페이지가 `domainA.com`에서 로드됐을 때, 이 페이지의 JavaScript가 `domainB.com`에서 리소스를 가져오려고 시도하면 크로스 오리진 오류가 발생합니다.

2. 이 오류는 개발자 도구의 콘솔에 메시지로 표시되며, 메시지에는 해당 요청이 동일 출처 정책을 위반했다는 내용이 포함됩니다.

### 해결 방법: CORS (Cross-Origin Resource Sharing)

- **CORS**는 서버 측에서 헤더 설정을 통해 선택적으로 동일 출처 정책을 완화하거나 해제할 수 있게 해주는 기술입니다.
- 서버는 `Access-Control-Allow-Origin` 헤더를 포함하는 응답을 보냄으로써, 특정 출처나 모든 출처에서의 리소스 요청을 허용할 수 있습니다.
  
```markdown
예제:
```
```
Access-Control-Allow-Origin: *
```
- 위의 헤더는 모든 출처로부터의 요청을 허용함을 의미합니다.

### 주의사항

- 모든 출처를 허용하는 설정은 보안 위험이 있을 수 있으므로, 필요한 출처만 허용하도록 설정하는 것이 좋습니다.
- 민감한 데이터를 다루는 애플리케이션에서는 특히 조심해야 합니다.
